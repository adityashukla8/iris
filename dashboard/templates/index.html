Errors propagate silently through agent pipelines
A single upstream hallucination cascades through every downstream agent with no circuit breaker. Without a supervisor, multi-agent clinical pipelines have no mechanism to stop compounding errors.

No self-improvement loop for clinical AI

Architecture

A Closed-Loop Safety System

IRIS operates as a continuous supervisor — evaluating every AI output in real time, detecting failure patterns from observability data, and autonomously improving clinical prompts through a validated self-healing loop.

01
Ingest IrisEvent
Clinical AI agents submit every output as a structured IrisEvent via POST /event. Includes the query, output text, retrieved patient context, and surgical phase.
FastAPIREST API
02
7-Evaluator Safety Check
Safety Evaluator runs 7 parallel clinical checks: dosage boundaries, hallucinations, drug-drug interactions, allergy contraindications, attribution, context gaps, and surgical phase alignment.
GeminiADK ToolsLLM-as-Judge
03
Observe to Phoenix
Every evaluation — including reasoning chain (CoT steps) and self-assessed confidence — is logged to Arize Phoenix as an OTel span annotation via OpenInference instrumentation.
OpenInferenceOTelPhoenix Cloud
04
Pattern Detection via MCP
Pattern Detector queries Arize Phoenix via MCP (get-spans, get-span-annotations) to identify recurring failure clusters across recent traces.
Phoenix MCPget-spans
05
Diagnose & Log via MCP
Self-Healer reads worst-performing spans, retrieves the current prompt version history, logs failure examples to a Phoenix dataset, and produces a structured HealingDiagnosis.
get-latest-promptadd-dataset-examples
06
Mutate, Validate & Deploy
TextGrad generates a candidate prompt mutation from the failure analysis. Counterfactual validation gates improvement. The winning version is deployed to Phoenix with an optional human approval gate.
TextGradApproval GatePhoenix Deploy
Multi-Agent System

Five Specialized Agents.
One Safety Guarantee.

Each agent is an ADK LlmAgent with purpose-built tools and MCP integrations. The orchestrator delegates to sub-agents via LLM-driven routing — no hardcoded pipelines.

Root Orchestrator
iris_orchestrator
Receives all IrisEvents and SCAN_PATTERNS requests. Routes to sub-agents via LLM-driven delegation. Bridges MCP-diagnosed failures to the Python healing pipeline via a dedicated ADK tool.
run_healing_pipeline_tool4 sub-agents
Clinical Auditor
safety_evaluator
Runs 7 parallel clinical safety tools per event. Writes results with reasoning chain and confidence to Arize Phoenix as span annotations. Flags low-confidence results for human escalation.
dosage_boundarydrug_interactionallergy_check+4 more
Failure Cluster Analyst
pattern_detector
Queries Arize Phoenix via MCP to detect recurring failure clusters across recent spans. Triggers the self-healing loop when the hallucination rate exceeds the configured threshold.
