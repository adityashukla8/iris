Architecture

A Closed-Loop Safety System

IRIS operates as a continuous supervisor — evaluating every AI output in real time, detecting failure patterns from observability data, and autonomously improving clinical prompts through a validated self-healing loop.

Step 01
📥
Ingest IrisEvent
Clinical AI agents submit every output as a structured IrisEvent via POST /event. Includes the query, output text, retrieved patient context, and surgical phase.
FastAPI REST API
Step 02
🛡️
7-Evaluator Safety Check
The Safety Evaluator agent runs 7 parallel clinical checks: dosage boundaries, factual hallucinations, drug-drug interactions, allergy contraindications, attribution, context gaps, and surgical phase alignment.
Gemini ADK Tools LLM-as-Judge
Step 03
📡
Observe to Phoenix
Every evaluation — including reasoning chain (CoT steps) and self-assessed confidence — is logged to Arize Phoenix as an OTel span annotation via OpenInference instrumentation.
OpenInference OTel Phoenix Cloud
Step 04
🔍
Pattern Detection via MCP
The Pattern Detector agent queries Arize Phoenix via MCP (get-spans, get-span-annotations) to identify recurring failure clusters across recent traces.
Phoenix MCP get-spans
Step 05
🩺
Diagnose & Log via MCP
The Self-Healer reads worst-performing spans, retrieves the current prompt version history, and logs failure examples to a Phoenix dataset — producing a structured HealingDiagnosis.
get-latest-prompt add-dataset-examples
Step 06
🔧
Mutate, Validate & Deploy
TextGrad generates a candidate prompt mutation from the failure analysis. Counterfactual evaluation validates improvement exceeds threshold. The winning version is deployed to Phoenix.
TextGrad Approval Gate Phoenix Deploy
Multi-Agent System

Five Specialized Agents.
One Safety Guarantee.

Each agent is an ADK LlmAgent with purpose-built tools and MCP integrations. The orchestrator delegates to sub-agents via LLM-driven routing — no hardcoded pipelines.

🧠
Root Orchestrator
iris_orchestrator
Receives all IrisEvents and SCAN_PATTERNS requests. Routes to sub-agents via LLM-driven delegation. Bridges MCP-diagnosed failures to the Python healing pipeline via a dedicated ADK tool.
run_healing_pipeline_tool 4 sub-agents
🛡️
Clinical Auditor
safety_evaluator
Runs 7 parallel clinical safety tools per event. Writes results with reasoning chain and confidence to Arize Phoenix as span annotations. Flags low-confidence results for human escalation.
dosage_boundary drug_interaction allergy_check +4 evaluators
📊
Failure Cluster Analyst
pattern_detector
Queries Arize Phoenix via MCP to detect recurring failure clusters across recent spans. Triggers the self-healing loop when the hallucination rate exceeds the configured threshold.
get-spans get-span-annotations list-traces Phoenix MCP
🔧
DIAGNOSE Phase
self_healer
Reads worst-performing spans from Phoenix, retrieves current clinical prompt version history, logs failure examples to a Phoenix dataset, and produces a HealingDiagnosis JSON for the mutation engine.
get-latest-prompt list-prompt-versions add-dataset-examples 10 MCP tools total
🚨
Notification Router
alert_dispatcher
Routes safety alerts by severity to the real-time dashboard SSE stream. INFO → feed update. WARNING → yellow badge. CRITICAL → red alert and human escalation queue.
push_dashboard_alert SSE stream
